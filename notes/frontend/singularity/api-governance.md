# API 治理与版本策略

> Singularity 的 API 稳定性承诺与版本管理规则。

---

## 一、版本策略

遵循 [Semantic Versioning](https://semver.org/)：

| 版本类型 | 含义                   |
| :------- | :--------------------- |
| `0.x.x`  | 开发阶段，API 可能变化 |
| `1.0.0`  | 稳定版，API 冻结       |
| `1.x.0`  | 新增功能，向后兼容     |
| `x.0.0`  | 重大变更，可能不兼容   |

---

## 二、API 稳定性承诺

### 核心 API（1.0 后冻结）

| API          | 状态    |
| :----------- | :------ |
| `atom()`     | ✅ 稳定 |
| `computed()` | ✅ 稳定 |
| `effect()`   | ✅ 稳定 |
| `batch()`    | ✅ 稳定 |
| `useAtom()`  | ✅ 稳定 |

### 扩展 API（可能变化）

| API              | 状态            |
| :--------------- | :-------------- |
| `atom.history()` | ⚠️ 开发模式专用 |
| `atom.restore()` | ⚠️ 开发模式专用 |

---

## 三、废弃流程

1. 标记 `@deprecated`
2. 在 2 个次版本后移除
3. 提供迁移指南
