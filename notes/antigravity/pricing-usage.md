## 配额机制

### 配额计算方式

配额基于 **"工作量"** 计算，而非简单的请求次数：

| 任务类型     | 配额消耗 |
| ------------ | -------- |
| 简单代码补全 | 低       |
| 单文件编辑   | 中       |
| 多文件重构   | 高       |
| 深度推理任务 | 最高     |

### 配额刷新

- **免费用户**：每周滚动配额，每 5 小时部分刷新
- **付费用户**：更大配额池，每 5 小时刷新

### 配额耗尽时

```
⚠️ Rate limit reached

配额不足时的表现：
- 响应速度变慢
- 排队等待时间增加
- 可能暂时降级到 Flash 模型
```

## 查看用量

### 在 Antigravity 中查看

```
设置 > Usage > View Current Usage
```

显示内容：

- 当前配额使用率
- 剩余配额
- 刷新倒计时
- 历史使用趋势

### 命令查看

```
/usage      # 查看当前用量
/quota      # 查看配额状态
/limits     # 查看限制详情
```

## 优化配额使用

### 1. 选择合适的模型

```
简单任务 → Gemini 3 Flash（配额消耗低）
复杂推理 → Gemini 3 Pro
超深度分析 → Deep Think（谨慎使用）
```

### 2. 使用 Planning Mode

先规划再执行，避免无效尝试：

```
【Planning Mode】
先分析这个需求，创建计划，不要开始编码
```

### 3. 提供精确上下文

减少 Agent 的搜索和分析工作：

```
参考 @src/components/Button.tsx 风格
创建 Card 组件
```

### 4. 批量处理

将相关任务合并为一个请求：

```
❌ 多次小请求
创建 Header 组件
...
创建 Footer 组件

✅ 一次批量请求
创建以下组件：Header, Footer, Sidebar
```

### 5. 使用缓存

相似任务利用已有上下文：

```
继续刚才的工作，添加暗色模式支持
```

## 常见问题

### Q: 免费版够用吗？

A: 对于个人学习和小型项目，免费版的周配额通常足够。如果你是全职开发者，建议考虑付费版。

### Q: 配额用完怎么办？

A:

1. 等待配额刷新（每 5 小时）
2. 升级到付费计划
3. 切换到配额消耗更低的模型（如 Flash）

### Q: 付费版有硬性限制吗？

A: 专业版和企业版没有硬性限制，但有公平使用政策防止滥用。

### Q: 可以查看详细用量记录吗？

A: 是的，在设置 > Usage 中可以查看详细的历史记录和趋势分析。
